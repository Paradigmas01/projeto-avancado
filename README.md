#Descrição do projeto 

## Avaliação do Divulga IF 
### Porque iremos fazer a avaliação do divulga IF?

Ao analisar a demanda por testes de software, identificamos o sistema [DivulgaIF](https://divulgaif.com.br/pt), desenvolvido nas disciplinas de Projeto Integrador e Construção de Software no terceiro ano de Engenharia de Software. Devido ao tempo limitado do ano letivo, o projeto foi concluído sem a devida implementação de testes unitários.

Diante disso, na disciplina de Projeto de Software Avançado, ministrada pelo professor [Hélio Kamakawa,](https://github.com/heliokamakawa) surge a oportunidade de iniciar uma etapa essencial do projeto: a implementação de testes unitários, visando melhorar a qualidade, confiabilidade e manutenção do sistema.

### Oque será desenvolvido 

Serão desenvolvidos testes unitários para o front-end do site DivulgaIF, com o objetivo de avaliar seu funcionamento e analisar aspectos de interação humano-computador, identificando possíveis problemas de usabilidade e falhas no sistema. A proposta busca suprir a ausência de testes no projeto, garantindo maior qualidade e confiabilidade. A construção será baseada em protocolos, artigos e documentos verificados, que orientarão a elaboração de prompts para utilização de IAs na geração e implementação dos testes.

## Etapas do processo 

- Clonar o repositório do divulga IF
- Gerar testes unitários do front-end utilizando inteligência artificial
- Criar uma branch para realizar seus comites
- Os Commits deve ser padronizados seguindo padrão de tipagem: feat, fix, docs, style, refactor, perf, test, build, ci, chore, revert, hotfix, wip, init, config, deps
- As branches para geração de testes unitários deverão seguir a seguinte descrição como exemplo: PSA-2026/(tipagem)/nome-da-branch
